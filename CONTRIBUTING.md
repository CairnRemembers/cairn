# Contributing to Cairn

Cairn is a solo-maintained, source-available project. **Issues and ideas are
very welcome; code contributions are read carefully and merged slowly.** Nothing
merges automatically — the maintainer reviews every line.

## The fastest way to help
- **Open an issue** for a bug, an idea, or a question — no code needed, all upside.
- **Planning a big change? Open an issue first.** A quick "is this wanted?" before
  you build it saves you sinking hours into something that won't merge — nothing
  lands here without a maintainer review.

## Pull requests
- Keep them small and focused; explain the *what* and the *why*.
- Cairn's base is **stdlib + numpy** — PRs that add NEW third-party runtime
  dependencies will not be merged.
- Every PR is reviewed line-by-line before merge.

## Contributor License Agreement (CLA)
Cairn is dual-licensed: **free for noncommercial use, with commercial licenses
sold by the maintainer** (this is what funds ongoing development). To keep that
possible, merged code contributions require a one-time **CLA** granting the
maintainer the right to include your contribution in Cairn, *including*
commercially-licensed versions. You keep the copyright to your own work — you're
just allowing it to ship inside Cairn. We'll point you to the CLA on your first
pull request.
